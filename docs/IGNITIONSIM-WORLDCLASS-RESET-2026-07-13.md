# IgnitionSim World-Class Reset

Date: July 13, 2026

## The Honest Baseline

The first quantitative pass found 80 certified articles averaging 63.3/100. The content library has useful raw material and averages 9.3 source URLs, but the public promise is ahead of the evidence on the page:

- 2,281 average editorial words
- 7.8 average unique images
- 0.9 real use/install images
- 0.2 curator action scenes
- 1.2 practical diagrams
- 52 of 80 certified guides currently grade D under the stricter world-class model

The strongest reference is the tactile/bass-shaker guide at 90/100. It works because it combines product proof, installed-use context, a practical system diagram, strong buying judgment, and a curator doing recognizable work. That is the repeatable pattern.

## The Revenue Objective

The horizon remains $10,000/month within a year. The path is not more low-proof inventory. It is a smaller number of pages that earn trust, convert high-intent readers, and create defensible topical clusters.

Target revenue mix at maturity:

| Channel | Monthly target | What must exist first |
| --- | ---: | --- |
| Specialty and Amazon affiliate | $5,000 | High-AOV buyer maps, exact product links, room/rig fit tools |
| Display advertising | $2,000 | Sustained qualified sessions and strong engagement |
| Newsletter sponsorship | $1,500 | A useful weekly briefing with measurable opens and clicks |
| Direct brand placements | $1,500 | Trust, disclosure discipline, and category authority |

## 30-Day Rescue

1. Repair the highest-AOV Tier A pages before adding volume: flagship direct drive, active pedals, premium launch monitors, luxury golf rooms, displays/VR, and console direct drive.
2. Give every rescued page at least one real human/use or installation photograph, one large product-proof sequence, one practical diagram, and one curator action scene.
3. Replace generic product grids and repeated cutouts with wide, inspectable evidence. Products remain real; generated work supplies only atmosphere, annotation, and the fictional crew.
4. Add contextual Crew Chat across articles and bays. It should answer the page's central question, remember the reader's bay, and recommend the next certified guide.
5. Make authorship transparent: Robert Pruitt carries editorial responsibility; curators are labeled AI-assisted bench personas.
6. Verify every commerce link. Amazon links must resolve to an exact `/dp/ASIN` page with `tag=ignitionsim-20`; specialty links should be labeled clearly.

## 60-Day Authority Build

Build four complete revenue clusters rather than scattering articles:

- Racing control chain: cockpit stiffness -> pedals -> wheelbase -> display -> haptics -> power/USB.
- Golf room chain: room measurement -> launch monitor -> mat/screen -> projector -> PC/software -> commissioning.
- Flight control chain: yoke/stick -> throttle -> pedals -> mounts -> display/VR -> USB and profiles.
- Space control chain: HOSAS/HOTAS decision -> stick ecosystem -> mounts -> bindings -> head tracking -> game-specific profiles.

Each hub should expose one beginner map, one serious-builder map, one premium path, a compatibility diagram, and a visible "buy once, cry once" recommendation with caveats.

## 90-Day Conversion Layer

- Launch one genuinely useful calculator per mature cluster: golf room-fit/throw calculator first, then racing cockpit/display fit, then USB/power planner.
- Add comparison events to analytics: verdict expansion, product proof viewed, source opened, exact product click, specialty-vendor click, and newsletter signup by article and bay.
- Publish a weekly Workbench Drop built from current product changes, firmware warnings, deals, and strong forum questions. It should route readers back into cornerstone pages instead of becoming disposable news.
- Use Search Console and Bing AI Performance to refresh pages earning impressions or citations but weak clicks. Rewrite titles and answer blocks from real query language, not generic keyword variants.

## The Weekly Operating Loop

1. Run `npm run audit:quality` and take the highest-revenue page below 85.
2. Verify current specs, compatibility, firmware, availability, and buyer complaints.
3. Acquire official product/use/install media and record the source in the image lexicon.
4. Add the missing decision layer, visual evidence, and curator action.
5. Run gold audit, quality audit, build, mobile/desktop visual QA, direct-link QA, and live deployment verification.
6. Measure clicks, scroll depth, Crew Chat use, newsletter assists, and revenue by article after publication.

The rule is simple: a page is not finished when it contains products. It is finished when a builder can picture the result, understand the work, make the decision, and reach the exact thing worth buying.
